# arqParse GUI - Графический Интерфейс

Минималистичный кроссплатформенный графический интерфейс, работающий на **Linux** и **Windows**.

## 🚀 Быстрый Старт

### Способ 1: Удобные скрипты (рекомендуется)

**Linux/Mac:**
```bash
./run_gui.sh
```

**Windows:**
```cmd
run_gui.bat
```

✨ Скрипты автоматически:
- Проверяют Python и Tkinter
- Создают виртуальное окружение (venv)
- Устанавливают зависимости
- Запускают GUI

### Способ 2: Прямой запуск

```bash
python main.py --gui
```

## 📋 Возможности

### Основные действия
- **📥 Скачать конфиги** — загрузить свежие VPN конфиги с источников
- **⚡ Тестировать все** — запустить полное тестирование всех задач
- **📂 Открыть результаты** — открыть папку с результатами в файловом менеджере
- **⏹ Остановить** — остановить текущую операцию
- **🗑 Очистить лог** — очистить окно логирования

### Быстрые действия по задачам
Для каждой задачи доступна кнопка отдельного тестирования:
- 🧪 Тест: Base VPN
- 🧪 Тест: Bypass VPN  
- 🧪 Тест: Telegram MTProto

## 🖥️ Системные требования

- Python 3.7+
- Tkinter (встроен в большинство дистрибутивов Python)

### Проверка Tkinter

```bash
python -m tkinter
```

Если окно открылось — все в порядке.

## � Виртуальное окружение (venv)

Скрипты `run_gui.sh` и `run_gui.bat` автоматически управляют виртуальным окружением:

### Что происходит при первом запуске:

1. **Проверка Python** — убедиться что установлен Python 3.7+
2. **Проверка Tkinter** — убедиться что Tkinter доступен
3. **Создание venv** — создается папка `venv/` с изолированным окружением
4. **Установка зависимостей** — устанавливаются пакеты из `requirements.txt`
5. **Запуск GUI** — запускается GUI через Python из venv

### Как это выглядит на экране Linux/Mac:

```

→ Создание виртуального окружения...

→ Обновляю pip...
→ Проверяю зависимости...
✓ Все зависимости уже установлены
```

### Папка venv

После первого запуска появляется папка `venv/`:

```
arqParse/
├── venv/                    # ← Виртуальное окружение
│   ├── bin/                 # Бинарники на Linux/Mac
│   ├── Scripts/             # Бинарники на Windows
│   ├── lib/                 # Python пакеты
│   └── pyvenv.cfg
├── run_gui.sh              # Скрипт запуска
├── gui.py
└── main.py
```

Можно безопасно удалить `venv/` — он будет пересоздан при следующем запуске.

## �📝 Примеры использования

### Скачивание конфигов
1. Нажмите **📥 Скачать конфиги**
2. Следите за прогрессом в окне логирования
3. При готовности появится сообщение об успехе

### Тестирование
1. Нажмите **⚡ Тестировать все** для полного теста
   или выберите конкретную задачу из быстрых действий
2. Монитор показывает:
   - ✓ — рабочие конфиги
   - ✗ — нерабочие конфиги  
   - Текущий прогресс

### Просмотр результатов
1. Нажмите **📂 Открыть результаты**
2. Откроется папка `results/` в файловом менеджере
3. Результаты сохранены в:
   - `top_vpn.txt` — лучшие VPN конфиги
   - `top_bypass.txt` — обходные конфиги
   - `top_MTProto.txt` — Telegram прокси
   - `all_top_vpn.txt` — объединенные конфиги

## 🐛 Решение проблем

### GUI не запускается

```bash
# Проверьте Python версию
python --version  # Должна быть 3.7+

# Попробуйте установить Tkinter вручную
# Ubuntu/Debian
sudo apt install python3-tk

# Fedora/CentOS
sudo yum install python3-tkinter

# macOS (если не установлено)
brew install python-tk@3.11  # для Python 3.11
```

### Конфиги не скачиваются

- Проверьте интернет соединение
- Убедитесь, что DNS работает корректно
- Проверьте файерволл

### Тестирование не работает

- Убедитесь, что `bin/xray` существует и имеет права на выполнение
- Для Linux: `chmod +x bin/xray`
- Проверьте наличие доступа в интернет

## 📌 Комбинированное использование

Вы можете использовать GUI и консоль вместе:

```bash
# Консоль для автоматизации
python main.py --force --no-ui

# GUI для интерактивной работы  
python main.py --gui

# Консоль с прокси
python main.py --proxy socks5://localhost:1080
```

## 💡 Советы

1. **Первый запуск** — используйте `--force` для свежего скачивания
2. **Большое количество конфигов** — увеличьте timeout в коде если нужно
3. **Сохранение результатов** — результаты сохраняются автоматически
4. **Логирование** — копируйте текст из логов для отладки
